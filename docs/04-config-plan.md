# 04 配置管理文档1

## 1. 配置项范围
- README
- 项目启动说明
- WBS 文档
- 进度计划文档
- 总结报告
- 示例代码或资源文件
- 配置管理方案文档
- 冲突测试文档
- src 项目代码目录
- test 测试目录
- assets 资源目录

## 2. 分支策略
- **main**：主分支，存放最终稳定版本，禁止直接提交
- **dev**：开发汇总分支，用于整合所有功能内容
- **feature-***：功能分支，用于单独开发文档与目录

## 3. 命名约定
- **文档命名**：使用 `01-xxx.md` 数字序号格式，清晰统一
- **分支命名**：功能分支以 `feature-模块名` 格式命名
- **提交说明**：采用 `类型: 描述内容` 格式
  - `docs:` 文档修改
  - `feat:` 新增目录/文件
  - `fix:` 修复错误、解决冲突

## 4. 合并规则
- **合并负责人**：项目经理 **邱梦楠**
- **合并时机**：单个任务完成 → 合并到 dev；全部完成 → dev 合并到 main
- **冲突处理**：
  1. 先拉取最新版本
  2. 手动删除冲突标记，保留正确内容
  3. 提交解决后的文件
  4. 完成合并并推送

## 5. 版本留痕
- 每个任务完成后必须提交，保留阶段性记录
- 所有提交使用规范说明，便于追溯修改人、修改时间、修改内容
- 不删除任何分支，保留完整开发历史
- 全部内容同步到 GitHub 远程仓库，永久留痕
